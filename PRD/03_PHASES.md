# MallBook (쇼핑몰 자산관리) -- Phase 분리 계획

> 한 번에 다 만들면 복잡해져서 품질이 떨어집니다.
> Phase별로 나눠서 각각 "진짜 동작하는 제품"을 만듭니다.

---

## Phase 1: MVP (3~4주)

### 목표
앱을 더블클릭으로 실행하고, 구글 로그인 후 거래를 입력하면 로컬 DB 저장 + 구글시트 자동 반영까지 되는 "일단 돌아가는" 제품.

### 기능
- [ ] 앱 설치/실행 (Windows .exe, macOS .dmg)
- [ ] 구글 OAuth 로그인 (구글시트 권한 포함)
- [ ] 사용자 초기 설정 (사업자번호, 과세유형, 연동 구글시트 선택)
- [ ] Counterparty CRUD (삼촌/공급업체/고객 등록·수정)
- [ ] Category 기본 세트 제공 (상품매입/임대/운송/기타) + 커스텀 추가
- [ ] Transaction CRUD (구매/판매/지출 입력·수정·삭제)
- [ ] 구글시트 단방향 쓰기 (Transaction 입력 시 시트에 행 추가)
- [ ] 홈 대시보드 (이번 달 총 매출·총 지출·거래 건수)
- [ ] 거래 목록 화면 (필터: 기간/타입/거래처/분류)

### 데이터
- User, Counterparty, Category, Transaction

### 인증
- 구글 OAuth (Google Sheets API 스코프 포함)

### "진짜 제품" 체크리스트
- [ ] 실제 SQLite DB 연결 (목업 데이터 X)
- [ ] 실제 구글 OAuth (하드코딩된 토큰 X)
- [ ] 설치 파일(.exe/.dmg)을 다른 PC에서 설치해서 실행 가능
- [ ] 터미널 명령 없이 아이콘 더블클릭으로 사용 가능
- [ ] 구글시트에 실제 행이 쌓이는지 검증

## Phase 2: 세금 & 현금흐름 (3주)

### 전제 조건
- Phase 1이 안정적으로 배포(설치 파일 배포)된 상태
- 최소 50건 이상 Transaction 데이터가 쌓여있음

### 목표
"세금 얼마 돌려받지?" "다음 주 들어올 돈 얼마?" 물음에 앱이 바로 답하는 상태.

### 기능
- [ ] TaxRecord 자동 생성 (Transaction 입력 시 공급가/부가세 자동 분리)
- [ ] 환급 대시보드 카드 (이번 분기 예상 환급액)
- [ ] CashflowItem 입력 (외상·미수금 예정일/금액 등록)
- [ ] 현금흐름 타임라인 (달력뷰로 예정 입출금 표시)
- [ ] 구글시트 양방향 동기화 (시트에서 수정한 내용 앱에 반영)
- [ ] 부가세 신고용 리포트 출력 (분기별 CSV/PDF)

### 추가 데이터
- TaxRecord, CashflowItem

### 통합 테스트
- Phase 1의 Transaction 입력 흐름에 세금·현금흐름 자동 생성이 추가돼도 기존 동작 유지되는지 확인
- 구글시트 양방향 동기화로 인한 충돌(앱↔시트 동시 수정) 처리 검증

## Phase 3: 분석 & 고도화 (2~3주)

### 전제 조건
- Phase 1 + 2가 안정적으로 운영 중
- 3개월 이상 실제 사용 데이터 축적

### 목표
"숫자만 보던 장부"를 "경영 인사이트 도구"로 승격.

### 기능
- [ ] 월/분기 손익 리포트 (매출 - 원가 - 비용 = 순이익)
- [ ] 카테고리별 지출 차트 (도넛/막대)
- [ ] 거래처별 거래 분석 ("삼촌과 1년간 거래 총액")
- [ ] 영수증 이미지 업로드 (Transaction에 사진 첨부)
- [ ] 월말 자동 리포트 (이메일/로컬 PDF)
- [ ] 데이터 백업/복원 (구글드라이브 자동 백업)

### 주의사항
- 영수증 이미지는 로컬 SQLite가 아닌 파일시스템 별도 폴더에 저장 (DB 비대화 방지)
- 구글드라이브 API는 Phase 1 OAuth 범위에 추가 스코프 필요 — 재인증 흐름 고려
- 손익 차트는 외부 라이브러리(recharts 등) 번들 크기 주의
