## 1. 제품 개요

### 한 줄 요약
1인 쇼핑몰 사장이 매일의 구매·판매·세금·현금흐름을 **엑셀/구글시트와 동기화되는 데스크톱 앱**으로 한눈에 관리하는 자산관리 도구.

### 해결하는 문제
- "오늘 뭐 샀지? 삼촌한테 얼마 줬지? 세금 얼마 돌려받지?" 매번 엑셀·문자·메모장 왔다갔다 하며 잃어버리는 거래 내역.
- 부가세 매입/매출·환급 금액을 직접 계산하느라 세무 신고철마다 발생하는 스트레스.
- 외상·미수금 등 "앞으로 나갈 돈, 들어올 돈" 예측이 안 돼 자금 계획을 못 세움.

### 핵심 가치
- **기존 엑셀 자료를 버리지 않음** — 구글시트와 양방향 연동해 익숙한 엑셀 그대로 활용.
- **세금 자동 계산** — 거래 한 줄 입력하면 부가세 매입/매출·환급 가능액이 자동으로 따라옴.
- **터미널 몰라도 됨** — 데스크톱 앱(더블클릭 설치)이라 가족·지인도 바로 사용 가능.
- **100% 무료 + 내 PC 저장** — 서버비 0원, 데이터는 내 컴퓨터의 SQLite 파일에 안전 보관.

## 2. 사용자

### 주요 사용자
- **누구**: 1인 쇼핑몰 사장 (본인), 그리고 필요 시 도와주는 가족·지인 2~3명
- **상황**: 매일 상품 구매·판매·비용 지출이 발생하고, 분기마다 부가세 신고를 해야 함. 터미널/코딩 지식 없음.
- **목표**: 거래 기록 → 세금 자동 계산 → 현금흐름 예측 → 월/분기 손익 확인까지 앱 하나로 해결.

### 사용자 시나리오
1. 사장이 아침에 앱을 더블클릭해서 열고
2. "삼촌한테 상품 매입 30만원 (부가세 포함)" 거래를 입력하면
3. 자동으로 공급가액/부가세 분리, 환급 가능액 계산, 구글시트에 기록
4. 저녁에 대시보드를 열어 "이번 달 남은 외상값 얼마, 다음 주 들어올 미수금 얼마"를 확인
5. 분기말 부가세 신고 시 세금 리포트 한 장으로 제출

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름
```
앱 실행 -> 구글 로그인 -> 거래 입력 -> 자동 세금 계산 -> 구글시트 동기화 -> 대시보드 확인
```

### 상세 흐름
1. **앱 실행**: 바탕화면 아이콘 더블클릭 → MallBook 창이 열림
2. **구글 로그인(최초 1회)**: 구글 계정으로 로그인 → 연동할 구글시트 선택
3. **거래 입력**: "신규 거래" 버튼 → 날짜/타입(구매·판매·지출)/상대/금액/분류 입력
4. **자동 처리**: 부가세 자동 분리, 환급 가능 여부 판단, SQLite 저장, 구글시트 쓰기
5. **대시보드**: 홈 화면에서 "이번 달 매출·지출·예상 환급액·다음 입출금 3건" 확인
6. **리포트**: "리포트" 탭에서 월/분기 손익, 카테고리별 지출 차트 확인

## 5. 성공 기준

- [ ] 거래 입력 → 구글시트 반영까지 3초 이내
- [ ] 부가세 자동 계산 정확도 100% (수작업 대비)
- [ ] 앱 설치 후 첫 거래 입력까지 5분 이내 (비개발자 기준)
- [ ] 월 100건 이상 거래를 입력해도 대시보드 로딩 1초 이내
- [ ] Windows/Mac 모두에서 더블클릭 설치 가능

## 6. 안 만드는 것 (Out of Scope)

> 이 목록에 있는 건 Phase 1에서 만들지 않습니다.
> AI에게 코드를 시킬 때 이 목록을 함께 공유하세요.

- 자동 세무 신고 API 연동 -- 이유: 국세청 연동은 복잡/리스크 큼. 리포트 출력까지만.
- 재고 관리 -- 이유: 쇼핑몰 쇼핑몰 연동(카페24/스마트스토어)은 별개 프로젝트 범위.
- 다중 사용자 권한 관리(사장/직원 구분) -- 이유: 1인 사장 + 가족 몇 명 공유 수준은 단일 계정으로 충분.
- 자동 결제/송금 기능 -- 이유: 금융 API는 인증·보안 부담 큼. 기록만 관리.
- 모바일 앱 -- 이유: PC 중심이라 Phase 1 제외. 추후 반응형 웹 확장 고려.
- 실시간 다중 접속 편집 -- 이유: 구글시트로 충분. SQLite는 로컬 단일 사용자.
